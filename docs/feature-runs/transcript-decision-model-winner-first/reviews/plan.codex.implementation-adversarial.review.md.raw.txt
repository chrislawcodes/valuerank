## Findings
- **High:** The plan is under-scoped relative to its own blast radius. `cloud/apps/api/src/routes/export.ts`, `cloud/apps/api/src/graphql/types/transcript.ts`, and `cloud/apps/web/src/utils/transcriptDecisionModel.ts` are all listed as directly affected, but none are assigned to a slice. That leaves a clear path for storage/resolver semantics to change while export, schema, or shared decoding code keeps the old interpretation.
- **High:** The no-backfill decision is not safe for the stated goal of pooling A-first and B-first runs correctly. The plan assumes the existing compatibility path already interprets historical transcripts correctly, but it never proves that. If the old parse path is where the inversion lived, every report or export that includes pre-existing rows will remain wrong.
- **Medium-High:** The storage contract is internally inconsistent. The plan says `direction` is derived at read time, but it also stores winner-first metadata inside `decisionMetadata.summaryCache.summary` with a `decisionState` discriminator. It never defines whether `summary` is still human-readable text or now a structured object, and it never requires validation that `decisionState`, `favoredValueKey`, `strength`, and `presentationOrder` agree. Fresh but semantically inconsistent rows could therefore be accepted as authoritative.
- **Medium:** The canonical-order rule is too brittle. Using `definitionSnapshot.methodology.pair_key` plus `dimensions[0]/[1]` assumes pair arrays are always present, ordered, and stable across versions. The plan does not say what happens when a snapshot is malformed, reordered, or legacy, so a small data-shape drift could silently bring back the wrong winner mapping.

## Residual Risks
- **Medium:** Even if the backend cache is correct, only the summarize path and one UI card get explicit regression coverage. Any other transcript-decision reader not named in the slices can still diverge until it is audited.
- **Medium:** The neutral vs unknown split depends on every downstream aggregation preserving `decisionState`. If any analytics or export step collapses them back into a single bucket, the distinction this plan tries to preserve will be lost.

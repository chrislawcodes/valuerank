## Residual Risks
- Judge dispatch still needs explicit prompt-size and context-budget rules. The current checkpoint machinery already has max-context and max-total knobs, but the spec does not define judge-side equivalents.

- Merge tracking still needs write-once semantics for provenance. `refresh_delivery_snapshot()` rebuilds delivery data from scratch, so later refresh logic can overwrite merge metadata unless the implementation protects those fields.

- The migration script still depends on the target workflows’ actual round counters. If either blocked workflow has fewer than 3 adversarial rounds, the normal `judge` gate will refuse unless the migration path has its own bypass.

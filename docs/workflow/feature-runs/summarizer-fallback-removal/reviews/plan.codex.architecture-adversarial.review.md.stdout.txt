## Findings
- Medium [CODE-CONFIRMED] The proposed unresolvable-count definition will miss `refusal` transcripts. The worker can persist `refusal` deterministically from explicit refusals ([summarize_extract.py](/Users/chrislaw/valuerank/.claude/worktrees/practical-noyce-1db12d/cloud/workers/summarize_extract.py#L288), [summarize_extract.py](/Users/chrislaw/valuerank/.claude/worktrees/practical-noyce-1db12d/cloud/workers/summarize_extract.py#L347)), and downstream scoring code treats refusal as a non-scoring outcome ([aggregate-helpers.ts](/Users/chrislaw/valuerank/.claude/worktrees/practical-noyce-1db12d/cloud/apps/api/src/services/analysis/aggregate/aggregate-helpers.ts#L43), [variance.ts](/Users/chrislaw/valuerank/.claude/worktrees/practical-noyce-1db12d/cloud/apps/api/src/services/analysis/aggregate/variance.ts#L40)). If the banner is meant to flag transcripts that still need human adjudication, this definition undercounts them.

- Medium [CODE-CONFIRMED] Removing `classify_decision_with_llm` from `summarize.py` breaks the current test and monkeypatch contract. The summarize tests still patch `summarize.classify_decision_with_llm` in several places ([test_summarize.py](/Users/chrislaw/valuerank/.claude/worktrees/practical-noyce-1db12d/cloud/workers/tests/test_summarize.py#L583), plus later fallback-path tests), so updating only assertions is not enough; the patch target itself must be rewritten or preserved.

- Medium [CODE-CONFIRMED] The new MCP `unresolvable` payload can push `get_run_summary` past the existing token budget. `get_run_summary` is capped at 5KB and only truncates `insights` and `llmSummary` today ([get-run-summary.ts](/Users/chrislaw/valuerank/.claude/worktrees/practical-noyce-1db12d/cloud/apps/api/src/mcp/tools/get-run-summary.ts#L35), [get-run-summary.ts](/Users/chrislaw/valuerank/.claude/worktrees/practical-noyce-1db12d/cloud/apps/api/src/mcp/tools/get-run-summary.ts#L102)), while `RunSummary` is expanded in the formatter layer ([formatters.ts](/Users/chrislaw/valuerank/.claude/worktrees/practical-noyce-1db12d/cloud/apps/api/src/services/mcp/formatters.ts#L39)). Adding a per-model breakdown without budget-aware pruning risks truncating more important summary fields on larger runs.

## Residual Risks
- [UNVERIFIED] The plan does not say whether `parseClass='unparseable'` should count as unresolvable. That class exists elsewhere in the resolver and shadow-validation code, so it may matter if any persisted transcript metadata uses it.

- [UNVERIFIED] The `decision_code_source IS DISTINCT FROM 'manual'` filter may still count legacy manual overrides if older rows were written before `decision_code_source` was normalized. The provided code shows the modern manual path sets the column, but it does not prove older rows do.

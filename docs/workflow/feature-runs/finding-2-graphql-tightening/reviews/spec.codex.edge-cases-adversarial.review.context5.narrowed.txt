00456: ## Soft Delete Pattern
00457: 
00458: Three tables use soft delete via `deleted_at` timestamp:
00459: 
00460: - `definitions` - Preserves version history
00461: - `scenarios` - Preserves transcript references
00462: - `definition_tags` - Preserves tag associations
00463: 
00464: **Rules:**
00465: 1. Records are never physically deleted
00466: 2. **All queries must filter** `WHERE deleted_at IS NULL`
00467: 3. GraphQL resolvers apply this filter automatically
00468: 4. `deleted_at` is NOT exposed in the GraphQL API
00469: 5. Cascading: when parent is deleted, related records are also soft-deleted
00470: 
00471: ```typescript
00472: // Querying (always filter deleted)
00473: const definitions = await prisma.definition.findMany({
00474:   where: { deletedAt: null }
00475: });
00476: 
00477: // Deleting (set timestamp, don't delete)
00478: await prisma.definition.update({
00479:   where: { id },
00480:   data: { deletedAt: new Date() }
00481: });
00482: ```

00510: ## Differences from Original Design
00511: 
00512: The schema evolved from the [original database design](../preplanning/database-design.md):
00513: 
00514: | Addition | Reason |
00515: |----------|--------|
00516: | `LlmProvider`, `LlmModel` tables | Database-driven provider/model configuration |
00517: | `RunScenarioSelection` table | Track sampled scenarios per run |
00518: | `SystemSetting` table | Runtime configuration |
00519: | Summary fields on Transcript | Store decision codes and summaries |
00520: 
00521: | Simplification | Reason |
00522: |----------------|--------|
00523: | No `ltree` extension | Recursive CTEs sufficient for our queries |
00524: | No `created_by` on most tables | Single-tenant, less audit trail needed |
00525: | Simpler experiment schema | Feature deferred |
